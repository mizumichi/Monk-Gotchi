# Monk-Gotchi プロジェクト規約

このファイルはClaude Codeが毎セッション読み込む永続ルール。
プロジェクト固有の不変ルールのみ記載する。一時的な指示はプロンプト側で。

---

## プロジェクト概要

- **アプリ名**: Monk-Gotchi (T-QUEST)
- **目的**: テストステロン向上習慣化アプリ
- **想定ユーザー**: 個人開発者の友人数人(クローズド配布)
- **利用想定**: スマホからWebサイト/将来PWA化

### ゲームシステム（現行）
- 毎日のタスクをこなすと累計XPが貯まり、**7日サイクルで木が育つ**（`TreeDisplay` SVG）
- サイクル終了時に累計スコアで **3段階ランク**（low/mid/high）が決まり、**実の数**が変わる
- 「収穫する」アクションで `Harvest` レコード作成 → **果樹園（/orchard）** に実が累積
- 収穫時は光ってから実が地面まで落ちる演出（T-4 収穫アニメーション）

> **設計方針メモ**: 23種のキャラ進化は管理負荷が大きく廃止した。単一指標（累計スコア）＋木モデルへ簡素化したことで、調整箇所が1関数（`getTreeRank`）に集約できた。

## 技術スタック

### フロントエンド
- Next.js (App Router)
- TypeScript
- Tailwind CSS
- @aws-amplify/ui-react (認証UI)

### バックエンド (AWS Amplify Gen2)
- 認証: Amazon Cognito (メール+パスワード)
- API: AWS AppSync (GraphQL)
- DB: Amazon DynamoDB
- ホスティング: AWS Amplify Hosting

### 開発環境
- WSL2 + Ubuntu 24.04 LTS on Windows 11
- VS Code + WSL拡張
- Node.js v20 (nvm管理)
- パッケージマネージャ: pnpm

## 開発ルール

### 1. ファイル変更の原則

各セッション開始時のプロンプトで「触ってOK」「触らない」が指定される。
それに従う。指定がない場合は以下のデフォルトに従う:

- **既存の動作するコードは触らない**(理由なき改変禁止)
- **Amplifyスキーマ定義は慎重に**(マイグレーションが必要になる)

### 2. 触る前に確認すべきこと

- 不明点があれば実装前にユーザーに質問する(勝手に判断しない)

### 3. 動作確認の必須項目

実装完了時、以下を必ず確認:

- `pnpm build` で型エラーゼロ
- `pnpm dev` でアプリ起動
- ブラウザコンソールにエラーなし
- 既存機能が壊れていない

### 4. Git/コミット

- ブランチ: `main` で直接開発(個人開発のため)
- コミットメッセージ: 日本語OK、Phaseナンバー入り
  - 例: `feat: Phase E-1 - キャラステージ追加(卵+待機モーション)`
  - 例: `fix: Phase E-1 - sticky配置の修正`
- 1コミット = 1機能(細かく分ける)

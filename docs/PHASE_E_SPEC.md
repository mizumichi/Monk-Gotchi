## 3. たまごっち風背景の仕様

### 3-1. 構成要素

```
┌─────────────────────────┐
│  空ゾーン（上1/3）        │ ← 単色 + 雲1〜2個
│       ☁                 │
├─────────────────────────┤
│  プレイエリア（中1/3）    │ ← キャラがここに立つ
│      [キャラ]            │
├─────────────────────────┤
│  床ゾーン（下1/3）        │ ← ドット模様の地面
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
└─────────────────────────┘
```

### 3-2. デザイン方針

- **枠**: 初代たまごっち風の「液晶画面」を意識した角丸の太い枠
- **配色**: 既存の「ダーク+紫テーマ」と調和させつつレトロLCD感を出す
  - 候補1: 暗紫の枠 × 明るい緑がかった液晶背景
  - 候補2: モノクロLCD風（暗灰の枠 × 黄緑の背景）
  - 詳細は E-1 実装時に Claude Code が提案して採用
- **背景パーツ**: 雲1〜2個 / 地面のドット模様 / 装飾オブジェクト（小さな草・石等）1〜2個
- **すべてSVG**で実装、軽量

### 3-3. 重要な制約

- 背景は**動かない**（または最小限のループのみ。ここではスコープアウト）
- キャラの視認性を最優先。装飾過多にしない

## 4. アニメーション仕様

### 4-1. 待機モーション（全キャラ共通）

| 項目 | 仕様 |
|---|---|
| 動作 | ふわふわ上下に浮く + 不定期にまばたき |
| 浮遊 | `translateY(0 → -4px → 0)` を 2秒で sin ループ |
| まばたき | 目パーツの `scaleY(1 → 0 → 1)` を 0.15秒、3〜5秒間隔でランダムトリガー |
| 実装 | CSS `@keyframes` メイン、まばたきのみ JS（`setInterval` + `Math.random`） |

### 4-2. タスク完了リアクション（全キャラ共通）

| 項目 | 仕様 |
|---|---|
| トリガー | タスクをチェック（チェックON）するたび 1回 |
| 動作 | キャラが小さくジャンプ（`translateY(-8px)` → 戻る） |
| 時間 | 約 0.4秒 |
| 実装 | React state で一時的にクラス付与 → 500ms後に削除 |
| キューイング | 連続チェック時は前のアニメーションをキャンセルして最新を再生 |

**重要**: 数値入力タスク（`sleep_hours`）も「1タスク=1リアクション」とする。値の大小に関わらず1回。

### 4-3. 進化演出（ステージ移行時）

| フェーズ | 内容 | 時間 |
|---|---|---|
| ① 光 | キャラ全体が白くフラッシュ（`opacity: 0 → 1 → 0`） | 0.5秒 |
| ② 消滅 | 縮小しながら透明に消える（`scale(1 → 0)` + `opacity(1 → 0)`） | 0.4秒 |
| ③ 登場 | 新キャラが小さく現れ、拡大しながらバウンス | 0.6秒 |
| ④ 完了 | 待機モーション開始 + 既存の進化トースト表示 | — |

**合計約 1.5秒**。既存の進化判定ロジック（`useCharacter.ts`）が進化を検知したタイミングで発火する。

## 7. 技術方針

| 項目 | 方針 |
|---|---|
| 描画方式 | SVG（インラインTSX） |
| アニメーション | CSS `@keyframes`（`transform` / `opacity` のみ使用） |
| ライブラリ追加 | なし（既存のNext.js + Tailwindのみ） |
| ドット絵単位 | 32×32 グリッド設計、表示は3〜4倍拡大 |
| 既存への影響 | ゲームロジック・DB・既存UIは**一切変更しない** |
| パフォーマンス | レイアウト再計算を起こす CSS プロパティは禁止（width/height変更等） |

## 11. 設計判断の根拠

| 判断 | 理由 |
|---|---|
| 既存アイコンを残す | バグ切り分けを容易にする。Phase Eの実装が壊れても既存UIで動作確認できる |
| カテゴリ別アニメは別フェーズ | 5カテゴリ × 23キャラ = 115パターンは現時点で過剰。共通モーションで土台を作る |
| 待機・タスク完了は全キャラ共通 | キャラ個別の動きは Phase G 以降の楽しみとして温存 |
| E-3で堕落・怠惰王を選択 | サボりルートは「+1日連打」で完走できテスト最速 |
| sticky で固定 | スマホでタスクスクロール中もキャラが見える「たまごっち感」を確保 |
| SVG＋CSS のみ | ライブラリ追加なし、Next.js環境への影響ゼロ |

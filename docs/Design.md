# Monk-Gotchi デザインブリーフ（Claude Design 用）

> Claude Design の最初のチャットにこのブリーフを貼り、あわせて GitHub リポジトリ「Monk-Gotchi」を指定し、各画面のスクリーンショットを添付すると、世界観を保ったまま案を出してもらいやすい。

---

## このアプリについて
Monk-Gotchi は、健康習慣化アプリ。5カテゴリ（筋トレ／睡眠／栄養／日光・環境／精神)の日々のタスクをこなして累計スコアを貯めると、木が7日サイクルでなめらかに育ち、7日目に「いまいち／普通／だいぶ良い」の3段階で実がなる。収穫した実は果樹園に木箱で貯まっていく。友人数人に配る個人開発アプリ。

## 渡し方（推奨）
- GitHub リポジトリ「Monk-Gotchi」を指定 → 既存の Tailwind 設定・色・コンポーネントをデザインシステムとして読み込ませる。
- ダッシュボードの現状スクリーンショットを添付。
- このブリーフをチャットに貼る。
- まずは1画面（ダッシュボード)だけで小さく試す。

---

## 全体トーン・世界観
- ベースは温かみのあるタスク管理。木と果樹園は絵本風のなめらかな質感。
- Finch のような優しさ。「罰しない・楽しく続ける」設計思想。
- 友人向けでカジュアル。20〜40代男性が、健康習慣を肩肘張らず続けられる雰囲気。

## 技術・表示の制約
- Next.js (App Router) + TypeScript + Tailwind CSS で実装する前提。**Tailwind のユーティリティで再現できる範囲**に収める。
- **モバイルの縦長画面ファースト**（スマホからの利用が主)。片手で操作しやすく。

## 既存アセット・カラー
- 木: 絵本風SVG（TreeDisplay)。幹 `#7A5230` / 葉・奥 `#3B6D11` / 葉・手前 `#5A9E2E` / 地面 `#639922`。
- 果樹園: 斜め上から見た木箱にリンゴ（CrateDisplay)。木箱 `#A86E3E` / 影線 `#7A4E29` / 内側 `#5E3C20`。
- 実 `#E24B4A`、収穫のきらめき（金) `#FFD24A`。
- 5カテゴリ: 筋トレ・睡眠・栄養・日光/環境・精神（既存のカテゴリ色を踏襲)。
- 既存テーマ色はリポジトリの Tailwind 設定を参照。

## 画面ごとのブリーフ

### 1. ダッシュボード（最優先）
- **goal**: 毎日開きたくなる温かみ。木の育ち具合が「今日の頑張り」と直結して感じられる。
- **layout**: 上に育成中の木 → 累計スコア・現在ランク・実数 → 5カテゴリのスコアバー → カテゴリタブ付きのタスク一覧 → ジャーナルの入口。
- **content**: 木（累計スコアで連続成長)、累計スコア／ランク／実数、5カテゴリのバー、タスク（チェック式＋睡眠の数値入力＋「⚠️回避」タスク)、Day表示。
- **audience**: スマホで使う20〜40代の友人。健康習慣を楽しく続けたい人。

### 2. 収穫モーメント
- **goal**: 7日間の達成感のピーク。光ってから実が根元に落ちる演出が映える舞台。
- **layout**: 中央に完成した木、収穫ボタン、収穫後にランクと実数。
- **content**: 木、光る演出、落ちる実、ランク（いまいち／普通／だいぶ良い)、実数。
- **audience**: 同上。

### 3. 果樹園（/orchard）
- **goal**: 続けるほど果樹園が充実する満足感。「貯まる嬉しさ」。
- **layout**: 上に合計実数＋収穫回数 → 収穫ごとの木箱カードのグリッド（新しい順)。
- **content**: 各カード＝斜め木箱＋リンゴ（量で盛り具合が変わる)＋日付＋累計ポイント＋実数。
- **audience**: 同上。

### 4. ログイン（付随）
- **goal**: 世界観に沿った入口。日本語。
- **layout**: シンプルな認証フォーム。
- **content**: メール＋パスワード（Amazon Cognito)。
- **audience**: 同上。

## 良くしたいポイント（探索してほしい方向）
- 配色とコントラストの洗練（温かみを保ちつつ、見やすく)。
- 余白と情報階層の整理（特にダッシュボードは要素が多い)。
- 木・果樹園の絵本風の質感を一段アップ。
- タスク一覧の見やすさ（5カテゴリ×多数タスク、回避タスクの区別)。
- モバイルでの片手操作のしやすさ。

## 使い方メモ
- ダッシュボード1画面で小さく試す 
- 出力は完成品ではなく下書き。チャットとインラインコメントで詰める。
- Claude に自分の出力のレビュー（コントラスト・階層・可読性)も頼める。
- 気に入った案は Claude Code へハンドオフして実装に渡す。
